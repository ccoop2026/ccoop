ccoop.in Unified Navigation Strategy

For the best UX and operational security, ccoop.in utilizes a unified app shell with role-based navigation.

1. The Single-App Approach

All users (Buyers, Hosts, Curators, Admins) log in through the same mobile web interface. Role-based access control (RBAC) determines which navigation elements and screens are visible.

2. Role-Based Navigation Maps

A. End-User (Buyer)

  • Primary Nav: Home (Drops/Retail) | Discover | My Campus | Profile.
  • Profile Menu: Order History, Wallet, Saved Items, Help, Settings.

B. Drop Host (Campus Staff)

  • Primary Nav: Home | Discover | Host Tools | Profile.
  • Host Tools: Incoming Scan, Dispatch Check, Stock Tally, Pickup Manifest.

C. Campus Curator (Store Manager)

  • Primary Nav: Home | Discover | Curator Dashboard | Profile.
  • Curator Dashboard: Campaign Creator, Sales Analytics, Inventory Management.

D. Global Admin (ccoop.in Team)

  • Primary Nav: Home | Discover | Admin Console | Profile.
  • Admin Console: Global Analytics, Partner Onboarding, Production Control.

3. Navigation Best Practices

  • The 'Role Switcher': For users with dual roles, the Profile section includes a prominent 'Switch to Admin View' toggle.
  • Universal Header: Always displays the 'Active Campus' and a search icon.
  • Persistent Back Logic: Navigation follows a 'Stack' model, allowing users to jump back to the Global Hub easily.
