Commerce & Fulfillment

The core logic governing the ccoop.in marketplace, pledge engine, and campus-led fulfillment. This section defines how users discover, commit to, and receive products.

The Two-Stage Pledge Engine

Phase 1 touch_app

Soft Launch (Pledge)

Zero-cost interest gathering to validate demand. Users 'Pledge' their interest without immediate payment. This enables demand forecasting before stock commitment.

Split-Order Handling

Logic for carts containing both 'Retail' and 'Drop' items. Retail items move to fulfillment immediately, while Drops wait for production cycles.

Pickup vs. Shipping

Campus Pickup is the default (free). Shipping is an available add-on for alumni or off-campus residents, calculated at checkout based on weight.
